# UniLinker v3.0 — P2P 设备互联平台设计文档

**日期**: 2026-05-31  
**协议**: MIT  
**定位**: 局域网设备互联平台 — 自动发现、建立 P2P 通道、插件化功能扩展  
**首个插件**: 屏幕投屏

---

## 1. 产品定位

### 1.1 核心理念

UniLinker 不是一个"投屏工具"，而是一个**局域网设备互联平台**。每台设备运行一个 uniLinker 实例，自动发现彼此并建立 P2P 连接。具体功能由插件提供，投屏只是第一个插件。

### 1.2 平台能力

| 能力 | 说明 | 提供者 |
|------|------|--------|
| 设备发现 | 局域网自动发现其他 uniLinker 实例 | Core |
| P2P 连接 | 设备间 WebRTC 加密直连通道 | Core |
| 配置文件 | 持久化用户设置 | Core |
| 插件宿主 | 加载/隔离/管理插件生命周期 | Core |
| 功能插件 | 投屏、文件传输、远程输入等 | Plugins |

### 1.3 未来插件规划

| 插件 | 简述 | 优先级 |
|------|------|--------|
| **屏幕投屏** (v3.0) | GPU 捕获 + H.264 硬编 + WebRTC 传输 | P0 当前 |
| **文件传输** | 拖拽传文件/文件夹，类似 AirDrop | P1 |
| **剪贴板同步** | 自动/手动同步剪贴板内容 | P1 |
| **远程输入** | 本机键鼠控制远端桌面 | P1 |
| **音频投送** | 把本机声音投到远端设备 | P2 |
| **控制端** | 手机远程控制电脑（聚合上述插件） | P2 |

### 1.4 关键指标

| 指标 | 目标 |
|------|------|
| 局域网端到端延迟 (投屏) | 4-10ms |
| 最高分辨率 | 2560x1600 @ 60fps |
| 画面质量 | H.264 硬件编码, 5-50Mbps CBR/VBR |
| 启动时间 | < 3秒 |
| 二进制大小 | < 25MB |
| 依赖 | 仅 Windows 10 1803+ 系统自带组件 |

### 2.2 核心层职责

| 模块 | 职责 | 对外接口 |
|------|------|----------|
| **Peer Mesh** | WebRTC 连接网格管理，多 PeerConnection 生命周期 | `IPeerMesh` |
| **Discovery** | mDNS 广播本机 + 发现局域网其他实例 | `IDeviceDiscovery` |

| **Plugin Loader** | 扫描/加载/隔离插件 Assembly，生命周期管理 | 内部使用 |
| **GUI Shell** | WebView2 窗口 + C#↔JS 桥 + 标签页容器 | 内部使用 |

### 2.3 P2P 互联模型

```
          局域网
    Desktop-A        Desktop-B        Laptop-C
    ┌──────────┐     ┌──────────┐     ┌──────────┐
    │uniLinker │     │uniLinker │     │uniLinker │
    │          │◄───►│          │◄───►│          │
    │ 📺 投屏   │     │ 📁 传文件 │     │ 👀 观看   │
    │ 📁 传文件 │     │ 👀 观看   │     │ 📺 投屏   │
    └──────────┘     └──────────┘     └──────────┘
    
    每个设备可同时运行多个插件，与多个 Peer 交互
```

### 2.4 连接拓扑

```
        A ←────→ B
        ↑  ↖   ↗ ↑
        │   ╲ ╱  │
        │   ╱ ╲  │
        ↓ ↙   ↘  ↓
        C ←────→ D
    
    全连接网格，每对 Peer 之间一条 WebRTC 连接
    信令通过已有节点转发（网状拓扑，无需中心服务器）
```

## 3. 插件系统

### 3.1 插件接口

```csharp
/// <summary>
/// 所有插件必须实现此接口。一个插件 = 一个功能。
/// 编译为独立 .dll，放在 plugins/ 目录下自动发现。
/// </summary>
public interface IPlugin
{
    /// <summary>全局唯一标识，如 "com.unilinker.screen-mirror"</summary>
    string Id { get; }

    /// <summary>显示名称，如 "屏幕投屏"</summary>
    string Name { get; }

    /// <summary>语义版本号</summary>
    string Version { get; }

    /// <summary>
    /// 插件声明的能力列表。
    /// 用于设备发现时的能力匹配。
    /// 如: ["screen-capture", "h264-encode", "video-stream"]
    /// </summary>
    string[] Capabilities { get; }

    /// <summary>
    /// 插件初始化。传入平台上下文，插件可以：
    /// - 注册 UI 面板 (context.UI.RegisterPanel)
    /// - 监听设备发现事件
    /// - 读写配置
    /// </summary>
    Task<bool> Initialize(IPluginContext context);

    /// <summary>
    /// 远端设备请求本插件能力时触发。
    /// 返回通信通道，平台负责传输。
    /// </summary>
    Task<IChannel?> OnPeerRequest(PeerInfo peer, string capability);

    /// <summary>
    /// 插件关闭时清理资源。
    /// </summary>
    Task Shutdown();
}
```

### 3.2 插件上下文（插件能用的所有平台服务）

```csharp
/// <summary>
/// 平台注入给插件的上下文。
/// 插件只依赖这个接口，不直接碰 WebRTC/mDNS/GUI 实现。
/// </summary>
public interface IPluginContext
{
    /// <summary>已建立的 Peer 连接网格</summary>
    IPeerMesh Peers { get; }

    /// <summary>局域网设备发现服务</summary>
    IDeviceDiscovery Discovery { get; }

    /// <summary>插件独立命名空间的配置存储</summary>
    IConfigStore Config { get; }

    /// <summary>GUI 扩展点</summary>
    IUIProvider UI { get; }

    /// <summary>插件自身元数据</summary>
    PluginInfo Self { get; }

    /// <summary>日志记录器</summary>
    IPluginLogger Logger { get; }
}
```

### 3.3 Peer 网格接口（插件跨设备通信的核心）

```csharp
/// <summary>
/// Peer 连接网格。插件通过此接口与其他设备通信。
/// </summary>
public interface IPeerMesh
{
    /// <summary>当前已连接的所有 Peer</summary>
    IReadOnlyList<PeerInfo> ConnectedPeers { get; }

    /// <summary>有新 Peer 连接时触发</summary>
    event Action<PeerInfo>? PeerConnected;

    /// <summary>Peer 断开时触发</summary>
    event Action<PeerInfo>? PeerDisconnected;

    /// <summary>
    /// 向指定 Peer 请求建立通道。
    /// capability 用于匹配远端插件（如 "screen-capture"）。
    /// 返回的 IChannel 可以是 MediaTrack（视频流）或 DataChannel（数据）。
    /// </summary>
    Task<IChannel?> CreateChannel(PeerInfo peer, string capability,
        ChannelOptions? options = null);

    /// <summary>
    /// 监听来自远端的通道请求。
    /// 当远端请求本机某个 capability 时触发。
    /// </summary>
    event Func<PeerInfo, string, Task<IChannel?>>? ChannelRequested;
}
```

### 3.4 通道抽象

```csharp
/// <summary>
/// Peer 间通信通道。可以是视频轨道或数据通道。
/// </summary>
public interface IChannel : IDisposable
{
    ChannelType Type { get; }           // MediaTrack / DataChannel
    PeerInfo RemotePeer { get; }
    string Capability { get; }
    ChannelState State { get; }

    // 仅 DataChannel 有效
    event Action<byte[]>? MessageReceived;
    Task SendAsync(byte[] data);

    // 仅 MediaTrack 有效
    event Action<EncodedPacket>? PacketReceived;   // 接收端
    Task SendPacketAsync(EncodedPacket packet);    // 发送端
}

public enum ChannelType { MediaTrack, DataChannel }
```

### 3.5 插件加载机制

```
启动时:
  1. 扫描 plugins/ 目录下所有 *.dll
  2. 用 AssemblyLoadContext 加载每个 DLL（插件间隔离）
  3. 查找实现了 IPlugin 的类型
  4. 调用 IPlugin.Initialize(context)
  5. 如果插件注册了 UI，通知前端渲染标签页

运行时:
  - 插件崩溃不影响其他插件（AppDomain 级隔离）
  - 未来可热加载/卸载（AssemblyLoadContext.Unload）
```

## 4. 屏幕投屏插件（首个实现）

### 4.1 插件定义

```csharp
public class ScreenMirrorPlugin : IPlugin
{
    public string Id => "com.unilinker.screen-mirror";
    public string Name => "屏幕投屏";
    public string Version => "1.0.0";
    public string[] Capabilities => new[] {
        "screen-capture",
        "h264-encode",
        "video-stream"
    };

    // 内部组件
    private ICapture? _capture;
    private IEncoder? _encoder;
    private readonly Dictionary<string, IChannel> _activeStreams = new();
}
```

### 4.2 数据流

```
[发送端]
Windows.Graphics.Capture (GPU 纹理)
  → ICapture.FrameCaptured 事件
  → IEncoder.Encode(frame)
  → IEncoder.PacketEncoded 事件
  → IChannel.SendPacketAsync(packet)
  → IPeerMesh → UDP → 局域网

[接收端]
UDP → IPeerMesh
  → IChannel.PacketReceived 事件
  → WebView2 → WebCodecs VideoDecoder (GPU 解码)
  → <video> 元素渲染
```

### 4.3 捕获接口（平台抽象）

```csharp
public interface ICapture : IDisposable
{
    event Action<CaptureFrame>? FrameCaptured;
    bool Start(int width, int height, int fps);
    void Stop();
    CaptureInfo GetInfo();
}

public record CaptureFrame(
    IntPtr D3dTexture,    // ID3D11Texture2D 指针（GPU 零拷贝）
    int Width,
    int Height,
    int Pitch,
    long TimestampUs);
```

**实现**: `WgcCapture` 使用 Windows.Graphics.Capture API。  
**未来**: Linux 实现 `PipeWireCapture`，macOS 实现 `SckCapture`。

### 4.4 编码器接口（可替换实现）

```csharp
public interface IEncoder : IDisposable
{
    event Action<EncodedPacket>? PacketEncoded;
    bool Initialize(int width, int height, int fps, int bitrateKbps);
    void Encode(CaptureFrame frame);
    EncoderInfo GetInfo();
}

public record EncodedPacket(
    byte[] Data,          // H.264 NAL 单元
    long TimestampUs,
    bool IsKeyFrame);
```

**默认实现**: `MfEncoder` 使用 Media Foundation SinkWriter（自动选 NVENC/QSV/AMF）。  
**未来**: `NvencEncoder` 直接调用 NVIDIA Video Codec SDK（方案 B）。

### 4.5 投屏连接流程

```
设备A (观看)                           设备B (投屏)
    │                                      │
    │  (mDNS 已发现 B，在设备列表中)         │
    │                                      │
    │  peerMesh.CreateChannel(              │
    │    peerB, "screen-capture")           │
    │  ──────────────────────────────────►  │
    │                                      │
    │                         plugin.OnPeerRequest(
    │                           peerA, "screen-capture")
    │                         → 启动 Capture + Encoder
    │                         → 返回 MediaTrack Channel
    │  ◄──────────────────────────────────  │
    │                                      │
    │  Channel.PacketReceived              │
    │  → 喂给 WebCodecs 解码               │
    │  → <video> 渲染                      │
    │                                      │
```

### 4.6 H.264 编码参数

```
profile = high
level = 5.1
bitrate = 10-50 Mbps CBR (根据分辨率自适应)
keyint = 120 (每 4 秒一个 IDR)
bframes = 0 (低延迟)
preset = speed (Media Foundation eAVEncH264VProfile_High)
rate_control = CBR
```

### 4.7 投屏延迟预算

| 环节 | 延迟 |
|------|------|
| WGC 捕获 | 1-3ms |
| Media Foundation H.264 编码 | 1-2ms |
| RTP 封包 | < 0.5ms |
| UDP 局域网传输 | < 1ms |
| RTP 解包 | < 0.5ms |
| WebCodecs 硬件解码 | 1-2ms |
| **总计** | **4-10ms** |

## 5. 技术栈

| 层 | 技术 | 许可证 | 原因 |
|----|------|--------|------|
| 语言 | C# 12 / .NET 9 | MIT | 现代 C#，NativeAOT 编译 |
| 编译 | NativeAOT | MIT | 单文件 .exe，无运行时依赖 |
| 屏幕捕获 | Windows.Graphics.Capture | Win SDK | GPU 零拷贝，Win10 1803+ |
| 硬件编码 | Media Foundation SinkWriter | Win SDK | 自动选 NVENC/QSV/AMD |
| WebRTC | SIPSorcery.WebRTC | MIT | 纯 C# 实现，无 libwebrtc |
| GUI | WebView2 | Win SDK | 复用 Web 前端，WebCodecs 解码 |
| 设备发现 | Zeroconf (mDNS) | MIT | 零配置局域网发现 |
| 分发 | 单个 .exe + plugins/ | MIT | ~20MB |

**关键决策**: 
- Windows 系统自带全部 GPU 捕获和编码能力，不依赖 FFmpeg 或外部 DLL
- NativeAOT 编译为单文件，用户下载即用
- 插件间通过 `AssemblyLoadContext` 隔离，可独立升级

### 7.2 前端插件注册

```javascript
// web/app.js — 插件 UI 注册 API
window.unilinker = {
  plugins: new Map(),

  // 插件调用此方法注册自己的标签页
  registerPlugin(pluginId, {
    title,        // 标签页标题
    icon,         // 标签页图标 (emoji)
    render,       // (container) => void — 首次渲染
    onActivate,   // () => void — 标签页激活时
    onDeactivate, // () => void — 标签页切走时
  }) {
    this.plugins.set(pluginId, { title, icon, render, onActivate, onDeactivate });
    this.renderTabBar();
  },

  // C# ↔ JS 通信
  callHost(method, ...args) {
    return window.chrome.webview.hostObjects.bridge[method](...args);
  }
};
```

    unilinker.callHost('StartDiscovery');
  },
  onDeactivate() {
    // 暂停刷新
  }

## 8. 设备发现与信令

### 8.1 mDNS 广播

每个 uniLinker 实例启动后：

1. 在 5353 端口启动 mDNS 服务
2. 注册服务 `_unilinker._tcp.local`，附带 TXT 记录：
   - `name`: 设备名称（用户可自定义）
   - `version`: uniLinker 版本
   - `port`: HTTP 端口（信令交换）
   - `capabilities`: 本设备支持的插件能力列表
   - `status`: 当前状态位掩码
3. 持续监听其他 `_unilinker._tcp.local` 实例
4. 设备上下线通过 mDNS TTL/Goodbye 自动处理

### 8.2 能力发现

```
设备A (有投屏插件)            设备B (有投屏 + 文件传输)
    │                              │
    │ mDNS TXT:                    │
    │ capabilities=                │
    │  screen-capture,             │
    │  h264-encode,                │
    │  video-stream                │
    │ ◄─────────────────────────── │
    │                              │
    │ A 知道 B 可以投屏              │
    │ → 显示 "👀 观看" 按钮         │
```

### 8.3 信令交换

WebRTC SDP/OFFER-ANSWER 通过 HTTP 交换：

```
A: POST http://B:port/signaling  { type: "offer", sdp: "...", capability: "screen-capture" }
B: 响应 { type: "answer", sdp: "..." }
```

ICE 候选通过同一端点交换。无需中心信令服务器。

## 11. 风险与回退

| 风险 | 概率 | 缓解方案 |
|------|------|----------|
| SIPSorcery H.264 WebRTC 支持不完整 | 中 | 回退 pion/webrtc + CGo（已验证可用） |
| Media Foundation 编码延迟不达标 | 低 | 切换到 NVIDIA Video Codec SDK（方案 B） |
| NativeAOT 不兼容 WinRT API (WGC) | 中 | 改为 JIT + 单文件发布，体积 ~50MB |
| WebView2 未安装 | 低 | Win10 1803+ 自带，检测并提示安装 |
| AssemblyLoadContext 插件卸载内存泄漏 | 低 | 先不支持热卸载，需重启程序 |
